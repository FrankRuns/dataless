Challenge: KPI Gaming Trap

Description

In this challenge, you’ll game a KPI system using synthetic data, then propose fixes to make it more robust. You’ll generate data to exploit flaws in a KPI metric, such as delivery on-time rate, and then suggest improvements to prevent gaming.

Video: Gaming KPIs with Synthetic Data

This video explores how synthetic data can be used to manipulate KPIs, revealing vulnerabilities in business metrics, and discusses strategies to design more robust KPIs.

Explanation

KPIs like delivery on-time rate can be gamed by manipulating data—for example, prioritizing easy deliveries to boost the metric while neglecting harder ones. In this challenge, you’ll generate synthetic data to exploit such a flaw, then propose fixes like adding penalties for neglected deliveries or balancing the metric with other KPIs. The goal is to understand how synthetic data can reveal system weaknesses and inform better metric design.

Hands-On Exercise

Generate a synthetic dataset of delivery records, including features like delivery time, difficulty, and priority. Use the Google Colab notebook below to game the on-time rate KPI, then propose fixes to make it more robust.
